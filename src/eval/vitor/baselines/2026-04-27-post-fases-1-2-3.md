## Gaps identificados

### Test quality (corrigíveis, não-blockers)

1. **Brittleness em substring assertions.** "decidido" vs "registrada", "assumption" vs "suposição". Mitigation: LLM judge avalia rubric semântica.
2. **Tool args matching estrito.** Quando seed gera id com timestamp, assertion com id literal falha. Mitigation: usar partial match em fields fixos só, ignorando ids.
3. **Setup incompleto em case-07.** Seedar persona + step data pra teste de confidence ter contexto.

### Vitor behavioral (need iteração de prompt)

1. **case-02:** Vitor preferiu live web_search em vez de consultar research log seedado. Indica que o prompt rule "consulte list_research antes" não tá forte o suficiente. **Fix sugerido:** adicionar regra explícita no prompt do brainstorm/prioritization sobre consultar log antes de pesquisa nova.

### Phase 4 dependencies (esperados)

1. `read_project_memory` — case-05
2. `read_session_memory(sessionId)` cross-session — case-06
3. `update_project_memory` + auto-compact hook — case-09
4. Compression de prompt > 3k tokens — case-08 (parcialmente)
