## Observações sobre o Vitor atual

Qualidades que NÃO regridem (continuam após o plano):

- **Briefing rico**: prompt já força BRIEF AUTOSSUFICIENTE, AC verificáveis, anti-padrões — [prompt.ts:139-281](../../../lib/agent/prompt.ts#L139-L281). Continua valendo.
- **Step coupling**: cada step tem prompt-mode próprio — produto, escopo, personas, brainstorm, prioritização, hipóteses, technical specs. Bom isolamento.
- **list_project_tasks** já existe pra evitar duplicação cross-session de **tasks** — mas não pra outras coisas (decisões, personas).

Gaps estruturais que o plano endereça:

- **Memória só vive em texto livre** (`sessionContext`) — sem tipo, sem status, sem id estável.
- **Sem confidence**: prompt geral diz "nao invente dados" mas não obriga etiqueta de incerteza nas saídas.
- **Sem aging**: não há sinal temporal — decisão de 2026-01 e de 2026-04 chegam no prompt iguais.
- **Sem cross-session**: cada session reseta o cérebro do Vitor.

## Como ler este baseline

- **Pred = fail** ≠ Vitor é ruim. É: a estrutura testada não existe, o teste é sobre **o que falta**.
- Quando uma fase do plano implementar a estrutura, o caso correspondente deve sair de `fail` pra `pass`.
- Se uma fase ship'a e o caso continua `fail`, o problema é prompt rule ou implementação — não estrutura. Investigar.
